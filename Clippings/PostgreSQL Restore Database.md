How to restore databases using psql
-----------------------------------

The `psql` tool allows you to restore the SQL script file generated by the `pg_dump`,Â  `pg_dumpall` or any other tools that generate compatible backed up files. By using theÂ `psqlÂ `tool, you can execute the entire script in the dump file.

To restore a full backup and ignore any error occurred during the restoration process, you use the following command:

```


`psql -U username -f backupfile.sql`

Code language: CSS (css)
```

If you want to stop restoring a database in case of errors, you add the `--set ON_ERROR_STOP=on option`:

```


`psql -U username --set ON_ERROR_STOP=on -f backupfile`

```

If you back up objects in a particular database, you can restore them using the following command:

```


`psql -U username -d database_name -f objects.sql`

```

How to restore databases using pg_restore
-----------------------------------------

Besides `psql`tool, you can useÂ  `pg_restore` program to restore databases backed up by theÂ  `pg_dump` or `pg_dumpall`tools. WithÂ  `pg_restore` program, you have various options for restoration databases, for example:

*   TheÂ  `pg_restore` allows you to perform parallel restores using the Â `-j` option to specify the number of threads for restoration. *   TheÂ  `pg_restore` also allows you to restore specific database objects in a backup file that contains the full database.
*   TheÂ  `pg_restore` can take a database-backed up in the older version and restore it in the newer version.
